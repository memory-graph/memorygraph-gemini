## ğŸ’¬ FAQ Responses

**Q: How is this different from Conductor?**
A: Complementary! Conductor excels at project-local context and task tracking. MemoryGraph adds cross-project, persistent memory. Better together.

**Q: Does this work with other AI tools?**
A: Yes! MemoryGraph uses MCP (Model Context Protocol), so it works with Claude Code CLI, Cursor, Windsurf, Continue.dev, and any MCP-compatible tool.

**Q: Is my data private?**
A: Absolutely. Default backend is local SQLite. Cloud sync is optional. You control your data.

**Q: What happens to my memories when I switch projects?**
A: They persist! That's the whole point. Your authentication pattern from Project A is searchable in Project B.

**Q: How do I store memories?**
A: Just use natural language: "Store this solution: ..." or "Remember that...". The AI handles the rest.

**Q: What if I have 1000s of memories?**
A: MemoryGraph uses semantic search with fuzzy matching. It finds relevant memories even with large knowledge graphs.
